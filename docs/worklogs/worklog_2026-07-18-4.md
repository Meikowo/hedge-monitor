# Worklog 2026-07-18（Extract Batch 事件主键冲突修复）

## 问题

最近一次 `Extract Batch (LLM)` 在抽取完成、运行 `build_events.py` 时失败：

`PostgREST 409 / 23505 / hedge_events_pkey / event_key=300252|2026|商品+外汇|p`

## 原因

未匹配计划的进展类公告会创建独立事件，但原来的主键只追加固定的 `|p`。同一公司、同一年度、同一类别出现多条此类公告时，会生成相同的 `event_key`；即使事件表已经清空，当前批次自身也会重复插入而触发主键冲突。

## 修复

1. 将未匹配进展事件的 key 改为 `...|p|{ann_id}`，利用公告主键生成稳定且可重跑的唯一 key。
2. 写库前增加重复 `event_key` 防护，若未来分组规则再次产生重复，会在写库前给出明确错误。
3. 本次失败发生在事件重建阶段，不需要重新支付或重新抽取已完成的 LLM 批次；修复部署后重跑事件构建即可。
4. 增加手动 `Build Events` workflow，使事件重建失败时可以单独修复派生层，不必为了重建事件再次消耗 LLM 配额。

## 下一步

合并修复后优先运行 `Build Events`；确认事件层恢复后，再继续运行 `Extract Batch (LLM)` 处理下一批 pending，避免重复消耗本次已经完成的 LLM 抽取。
