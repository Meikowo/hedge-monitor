## 2. M3 设计决策

用户确认正式版采用：

- A：shadcn/ui 风格的白底高密度数据工作台；
- C：点击事件后从右侧展开研究详情与证据时间线；
- 少装饰、细边界、强信息层级，页面只展示真实数据；
- 桌面端优先，同时适配移动端。

## 3. 本次实现

重写 `web/index.html`、`web/styles.css` 与 `web/app.js`：

1. 首页改为左侧工作区导航、四项真实指标、紧凑筛选器和高密度事件表。
2. 事件数据由固定 1,000 条上限改为分页读取全部记录，当前可完整读取 1,721 个事件。
3. 新增公司/代码/品种搜索，以及类别、审批层级、企业性质筛选。
4. 新增日期、公司、证据数排序和每页 50 条的客户端分页。
5. 点击事件后按 `event_key` 查询关联公告，在右侧展示事件字段、额度口径、程序回验状态、公告时间线、证据引文和 PDF 原文链接。
6. 公告原流改为首次切换时再分页加载，避免拖慢事件首屏。
7. 移动端将宽表转换为紧凑卡片式行，详情抽屉占满屏宽。
8. 修复旧页面遗留的乱码文本，所有前端文件统一保持 UTF-8。
9. 按 Supabase 当前公开 Key 规范，浏览器请求只发送 publishable `apikey`，不发送或暴露 service-role 密钥。
