# worklog_2026-07-17-5 - 抽取批次额度异常

## 1. 本次阶段

核验新一轮 `Extract Batch (LLM)` 完成后的数据库状态，并调查失败记录。

## 2. 数据结果

- `extractions`：708 条。
- `announcements` 状态：`extracted` 708、`pending` 2,678、`failed` 139、`skipped` 1。
- `quota_items`：1,023 条；其中 `amount_verified=true` 为 952 条，`quote_verified=true` 为 811 条。
- `hedge_events`：362 条；`event_members`：708 条。

## 3. 异常原因

139 条失败记录的共同原因是 MiniMax API 返回 HTTP 402：`insufficient_balance_error (1008)`。这不是重复写入，也不是 Supabase 连接失败；成功写入的抽取和事件数据仍然有效。

## 4. 决策

1. 暂停继续抽取，不立即运行 `retry_failed=true`。
2. 先在 MiniMax 控制台确认 token plan key 的实际余额/可用额度，并确认 GitHub Actions Secret 已使用该 key。
3. 额度确认后先运行 MiniMax probe；probe 成功后用 `limit=30` 重试一小批 failed，确认稳定后再恢复 `limit=300`。

## 5. 性能观察

当前抽取脚本按公告串行执行，每条包含 PDF 下载、正文解析、LLM 请求和失败退避；300 条接近两小时属于当前实现的预期表现。性能优化应等额度链路稳定后再做，避免同时引入并发、限流和费用变量。
